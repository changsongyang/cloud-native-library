Multi-cluster traffic failover with EastWest Gateways
With an EastWest gateway, any internal service can be made highly available with automated cross-cluster failover without publishing it for external access via an Ingress gateway. In this guide, you’ll: Deploy bookinfo application in one cluster cluster-1. You’ll deploy the same bookinfo application in a second cluster cluster-2. Cause the reviews, details or ratings service to fail in cluster-1, and observe that the bookinfo application has failed. Add the reviews, details and ratings services to an EastWest gateway. Repeat the failure scenario and observe that the application is not affected, and that internal traffic is routed to the services in cluster-2.
